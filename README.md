# TCP Chat Server Assignment

Python `socket` 모듈과 `threading` 모듈을 사용해 구현한 멀티 클라이언트 TCP 채팅 프로그램입니다.
기본 에코 서버 예제를 확장해 여러 사용자의 동시 접속, 닉네임 기반 채팅, 전체 메시지 브로드캐스트, 개인 메시지, 접속자 목록 조회, 서버 로그 기록을 지원합니다.

## 주요 기능

- TCP 기반 클라이언트-서버 통신
- 멀티스레드 방식의 다중 클라이언트 처리
- 닉네임 중복 검사
- 전체 채팅 메시지 브로드캐스트
- 개인 메시지 명령어 `/msg`
- 접속자 목록 조회 명령어 `/users`
- 명령어 도움말 `/help`
- 연결 종료 명령어 `/quit`
- 서버 실행 인자 `--host`, `--port`, `--backlog`, `--max-clients`, `--max-message-length`, `--log-file` 지원
- 클라이언트 실행 인자 `--host`, `--port`, `--nickname`, `--max-message-length` 지원
- 최대 접속자 수와 메시지 길이 제한
- `Ctrl+C` 서버 종료 처리와 종료 로그 기록
- 서버 이벤트와 채팅 내용을 `server.log`에 기록
- TCP 스트림 특성을 고려한 줄 단위 메시지 프로토콜 사용

## 파일 구성

- `server.py`: 채팅 서버
- `client.py`: 채팅 클라이언트
- `test_chat.py`: 서버/클라이언트 통신 자동 테스트
- `.gitignore`: 로그, 캐시, 가상환경, PDF 자료 제외 설정
- `server.log`: 서버 실행 중 생성되는 로그 파일

## 실행 방법

서버 실행:

```bash
python server.py
```

클라이언트 실행:

```bash
python client.py
```

닉네임과 포트를 인자로 지정해 실행할 수도 있습니다.

```bash
python server.py --host 127.0.0.1 --port 5000 --backlog 5 --max-clients 10
python client.py --host 127.0.0.1 --port 5000 --nickname Alice
```

여러 사용자의 동시 접속을 확인하려면 `python client.py`를 여러 터미널에서 실행하면 됩니다.

자동 테스트 실행:

```bash
python test_chat.py
```

## 명령어

| 명령어 | 설명 |
| --- | --- |
| `/help` | 사용 가능한 명령어를 보여줍니다. |
| `/users` | 현재 접속 중인 사용자 목록을 보여줍니다. |
| `/msg <nickname> <message>` | 특정 사용자에게 개인 메시지를 보냅니다. |
| `/quit` | 채팅 서버와의 연결을 종료합니다. |

## 동작 원리

1. 서버는 지정한 IP와 포트에 TCP 소켓을 바인딩하고 연결 요청을 기다립니다.
2. 클라이언트가 접속하면 서버는 새 스레드를 생성해 해당 클라이언트를 독립적으로 처리합니다.
3. 클라이언트는 접속 직후 닉네임을 전송하고, 서버는 중복 여부를 검사한 뒤 접속자 목록에 저장합니다.
4. 일반 메시지는 서버가 수신한 뒤 모든 접속자에게 브로드캐스트합니다.
5. `/msg` 명령어는 대상 닉네임을 찾아 해당 사용자에게만 메시지를 전송합니다.
6. 접속자 목록은 여러 스레드가 동시에 접근하므로 `threading.Lock`으로 보호합니다.
7. 서버는 최대 접속자 수와 메시지 길이를 검사해 기본적인 자원 사용을 제한합니다.
8. 서버는 접속, 퇴장, 일반 메시지, 개인 메시지를 로그 파일에 기록합니다.

## TCP 메시지 처리

TCP는 데이터를 바이트 스트림으로 전달하므로 한 번의 `send`가 한 번의 `recv`와 정확히 대응된다고 보장할 수 없습니다.
이 프로젝트는 각 메시지 끝에 줄바꿈 문자(`\n`)를 붙이고, 서버가 `readline()`으로 한 줄씩 읽는 방식으로 간단한 메시지 경계를 만들었습니다.

## 수업 개념 반영

| 수업 개념 | 코드 반영 |
| --- | --- |
| 클라이언트-서버 모델 | 서버는 `listen()`으로 대기하고, 클라이언트는 `connect()`로 접속합니다. |
| 프로세스 식별 | IP 주소와 포트 번호를 실행 인자로 지정할 수 있습니다. |
| 소켓 API 흐름 | 서버는 `bind()` → `listen()` → `accept()` 순서로 연결을 처리합니다. |
| 동시 접속 처리 | 클라이언트 연결마다 스레드를 생성합니다. |
| 공유 자원 보호 | 접속자 목록을 `threading.Lock`으로 보호합니다. |
| TCP 바이트 스트림 | 줄바꿈 기반 프로토콜로 메시지 경계를 구분합니다. |
| 서버 자원 관리 | `backlog`, 최대 접속자 수, 메시지 길이 제한을 적용했습니다. |

## 테스트 시나리오

1. 서버를 실행합니다.
2. 첫 번째 터미널에서 `python client.py --nickname Alice`를 실행합니다.
3. 두 번째 터미널에서 `python client.py --nickname Bob`을 실행합니다.
4. `Alice`가 `안녕하세요`를 입력하면 모든 클라이언트에 `[Alice] 안녕하세요`가 표시됩니다.
5. `Bob`이 `/users`를 입력하면 `Alice, Bob`이 표시됩니다.
6. `Alice`가 `/msg Bob 개인 메시지입니다`를 입력하면 Bob에게만 개인 메시지가 전달됩니다.
7. `/quit`를 입력하면 퇴장 알림이 다른 사용자에게 전달됩니다.

## 과제 설명 포인트

- TCP의 연결 지향 특성을 사용해 안정적인 클라이언트-서버 통신을 구현했습니다.
- 멀티스레드 구조로 여러 클라이언트를 동시에 처리했습니다.
- 공유 자원인 접속자 목록을 `Lock`으로 보호해 동시성 문제를 줄였습니다.
- 줄 단위 프로토콜을 적용해 TCP 스트림에서 메시지 경계를 관리했습니다.
- `listen(backlog)`와 최대 접속자 수 제한으로 서버 자원 관리 개념을 반영했습니다.
- 명령어 기반 기능을 추가해 단순 에코 서버보다 실제 채팅 프로그램에 가까운 구조로 확장했습니다.
