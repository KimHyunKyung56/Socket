# TCP Chat Server Assignment

Python socket module로 만든 멀티 클라이언트 TCP 채팅 프로그램입니다.
네트워크 수업 과제 제출을 위해 기본 에코 예제를 확장하여 여러 사용자의 동시 접속, 닉네임 기반 채팅, 서버 로그 기록 기능을 포함했습니다.

## 주요 기능

- TCP 기반 클라이언트-서버 통신
- 멀티스레드 방식의 다중 클라이언트 처리
- 닉네임 기반 채팅
- 입장 및 퇴장 알림 브로드캐스트
- 접속자 목록 조회 명령어 `/users`
- 클라이언트 종료 명령어 `/quit`
- 서버 이벤트 및 채팅 내용 `server.log` 저장

## 파일 구성

- `server.py`: 채팅 서버
- `client.py`: 채팅 클라이언트
- `server.log`: 서버 실행 중 생성되는 로그 파일

## 실행 방법

서버 실행:

```bash
python server.py
```

클라이언트 실행:

```bash
python client.py
```

여러 명이 접속하는 상황을 확인하려면 클라이언트를 여러 터미널에서 실행하면 됩니다.

## 사용 방법

1. 서버를 먼저 실행합니다.
2. 각 클라이언트는 접속 후 닉네임을 입력합니다.
3. 일반 메시지를 입력하면 모든 접속자에게 전달됩니다.
4. `/users`를 입력하면 현재 접속자 목록을 확인할 수 있습니다.
5. `/quit`를 입력하면 연결을 종료합니다.

## 동작 원리

1. 서버는 `127.0.0.1:5000`에 바인딩한 뒤 연결 요청을 기다립니다.
2. 클라이언트가 접속하면 서버는 새 스레드를 생성해 해당 클라이언트를 전담 처리합니다.
3. 클라이언트가 보낸 메시지는 서버에서 수신한 뒤 전체 사용자에게 브로드캐스트됩니다.
4. 서버는 접속, 종료, 채팅 메시지를 로그 파일에 기록합니다.

## 과제 설명에 적기 좋은 포인트

- TCP는 연결 지향 방식이므로 메시지 전달 신뢰성이 높습니다.
- 멀티스레드를 사용해 여러 클라이언트를 동시에 처리합니다.
- 공유 자원인 클라이언트 목록은 `Lock`으로 보호해 동시성 문제를 줄였습니다.
- 브로드캐스트 구조를 사용해 1:N 채팅 서비스 형태를 구현했습니다.

## 예시 시나리오

- 사용자 `Alice` 접속
- 사용자 `Bob` 접속
- `Alice`가 `안녕하세요` 입력
- 서버가 `[Alice] 안녕하세요`를 모든 클라이언트에 전송
- `Bob`이 `/users` 입력
- 서버가 현재 접속자 목록을 응답
